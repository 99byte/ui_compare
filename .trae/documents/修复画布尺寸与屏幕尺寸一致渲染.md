## 原因分析

- 画布内部尺寸已设置为屏幕大小，但 `<canvas>` 的 CSS 类为 `w-full h-[420px]`，固定高度 420px，导致视觉尺寸没有按照 `1260×2720` 更新。
- 父容器设置了 `overflow-hidden`，即使内部尺寸变大，外层会裁剪，肉眼看到仍近似 420px 高。

## 修改方案

### 前端画布 CSS 与状态
1. 添加屏幕尺寸状态：`designScreen`、`codeScreen`，来源于顶层 `attributes.bounds`。
2. 移除 `<canvas>` 上的固定高度类 `h-[420px]` 与 `w-full`，改为使用内联样式：`style={{ width: designScreen?.width, height: designScreen?.height }}`；代码侧同理。
3. 将容器 `overflow-hidden` 改为 `overflow-auto`，以便大尺寸画布可滚动查看完整屏幕。
4. 在文件上传后即刻解析并保存屏幕尺寸，初次绘制时按屏幕尺寸设置画布。

### 前端绘制逻辑
1. 保持当前 `drawComponentsOnCanvas` 以屏幕尺寸设置 `canvas.width/height`。
2. 绘制时使用原始坐标（`scale=1`），确保与 JSON 中像素一一对应。

### 后端
- 保持现状，无需改动：`/api/compare` 已返回 `screen:{width,height}`。

## 代码改动位置
- `frontend/src/app/page.tsx`：
  - 引入并维护 `designScreen/codeScreen`。
  - 更新 `<canvas>` JSX 样式为动态宽高，移除 `h-[420px]` 和 `w-full`。
  - 将画布外层容器类由 `overflow-hidden` 改为 `overflow-auto`。
  - 上传与比较后均按屏幕尺寸调用 `drawComponentsOnCanvas`。
